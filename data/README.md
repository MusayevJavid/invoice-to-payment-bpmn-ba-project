# Data

This folder contains the data dictionary for the Invoice-to-Payment Business Analysis project.

The data dictionary defines the key data fields used across the invoice validation, approval, payment, and reconciliation process. It helps clarify what information is required, where it is used, and why it is important for the process.

## File Included

| File                 | Purpose                                                                                                                                                       |
| -------------------- | ------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| `data-dictionary.md` | Defines the main data fields used in the Invoice-to-Payment process, including invoice, vendor, purchase order, payment, approval, and reconciliation fields. |

## Why This Section Matters

A Business Analyst should understand not only the process flow, but also the data that moves through the process.

For this project, the data dictionary supports:

* Clear understanding of invoice-related fields
* Better communication between business and IT teams
* More accurate functional requirements
* Improved validation rules
* Easier testing and acceptance criteria definition
* Better control over payment and reconciliation outcomes

## Main Data Areas

The Invoice-to-Payment process includes data related to:

* Vendor information
* Invoice details
* Purchase order reference
* Goods receipt or service confirmation
* Approval status
* Payment execution
* Bank confirmation
* Reconciliation status
* Exception handling

## Disclaimer

This is an anonymized portfolio project created for demonstration purposes. It does not include confidential company data or internal documentation from any employer.
